## Naming Conventions

**Files:**
- Python files: lowercase with underscores (`app.py`, `cpy.pyx`, `requirements.txt`)
- Arduino files: CamelCase for classes and modules (`WifiCaptive.cpp`, `Epd`, `EpaperManager`)
- Headers: lowercase `.h` extension (`epd7in3e.h`, `button.h`)
- Templates: lowercase with extension (`settings.html`)
- Configuration: lowercase with dot notation (`config.h`, `.gitattributes`)

**Directories:**
- Lowercase plural or singular based on content type:
  - `epd7in3e/` - module/subsystem (firmware package)
  - `templates/` - web resources
  - `CAD/` - uppercase for hardware domain
  - `.planning/codebase/` - hidden prefix for tool-generated content

**Functions:**
- Python: `snake_case` (e.g., `load_downloaded_images()`, `scale_img_in_memory()`, `sync_time_with_ntp()`)
- Arduino/C++: `camelCase` (e.g., `downloadImage()`, `setupWiFi()`, `drawImage()`)
- Cython: Mix of snake_case (public API) and camelCase/lowercase+underscore (internal)

**Variables:**
- Python global state: UPPERCASE for constants (e.g., `DEFAULT_CONFIG`, `ALLOWED_EXTENSIONS`), lowercase for globals (e.g., `current_config`, `albumname`, `url`)
- Python local: `snake_case` (e.g., `battery_voltage`, `image_order`, `sleep_duration`)
- Arduino: `camelCase` (e.g., `imageUrl`, `batteryVoltage`, `downloadPath`)

**Classes:**
- C++/Arduino: `PascalCase` (e.g., `EpaperManager`, `WifiCaptive`)
- Python: `PascalCase` (e.g., `ConfigFileHandler`, typical Python convention)

## Where to Add New Code

**New Feature (e.g., new image filter):**
- Image processing logic: Add function to `cpy.pyx` for Cython optimization, Python wrapper in `app.py`
- API endpoint: Add `@app.route()` decorator and handler function in `app.py`
- Firmware support: Add device communication in `epd7in3e/epd7in3e.ino`
- Configuration: Add keys to `DEFAULT_CONFIG` dict in `app.py` (lines 23-38), form fields to `templates/settings.html`

**New Component/Module (e.g., external API integration):**
- Server-side: Create new Python file in root directory (e.g., `immich_client.py`), import in `app.py`
- Firmware: Create new `.cpp/.h` pair in `epd7in3e/` directory following existing module structure
- Example: `filesystem.cpp/h` handles SPIFFS operations; `WifiCaptive.cpp/h` encapsulates WiFi portal

**Utilities (e.g., image helper functions):**
- Image utilities: Add to `cpy.pyx` (performance-critical) or create new `utils.py`
- Common constants: Add to top-level globals in `app.py` or firmware `config.h`
- Shared functions: Follow single-responsibility; extract to own file if >100 lines

**Web UI Enhancements:**
- Form fields: Add to `templates/settings.html` with corresponding form handling in `@app.route('/setting', methods=['POST'])`
- Validation: Add checks in `/setting` POST handler (lines 574-600)
- Template variables: Pass from Python via `render_template()` call (lines 617-620)
