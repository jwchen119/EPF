<objective>
Apply the four firmware-side battery optimizations identified in 10-RESEARCH.md to the XIAO ESP32-S3 firmware: (1) gate the 3 s USB-CDC boot delay so production deep-sleep wakeups use ~50 ms, (2) set CPU to 80 MHz and WiFi TX power to 8.5 dBm before connecting, (3) replace the hex-CSV decode loop in processImageData() with a direct binary readBytes() into the PSRAM frame buffer (must match the binary server from plan 10-01), and (4) isolate the BAT_ADC/ADC_EN GPIOs before deep sleep. A human-verify checkpoint confirms the device renders the binary image correctly on hardware.

Purpose: Eliminate ~0.056 mAh/cycle (boot delay) + ~0.034 mAh/cycle (CPU) + binary-decode speedup, and prevent ADC leakage in deep sleep. Combined with plan 10-01 this is ~4.4 mAh/day saved.
Output: Modified epd7in3e/epd7in3e.ino and epd7in3e/config.h; compile-clean; human-verified on device.

DEPENDS ON 10-01: the server now sends raw binary (application/octet-stream, 960000 bytes). This firmware MUST decode binary, not hex. Deploying this firmware against an old hex server (or vice-versa) corrupts the frame buffer (Pitfall 3). Server (10-01) and firmware (10-02) must ship together.
</objective>
